# VLM Judge 失分原因分析

本分析基于 `qwen / qwen3-vl-plus` 对 `design2code`、`flame`、`fullfront` 三组各 10 条样本的视觉 judge 结果，并人工查看了目标图/生成图对照。

## 总体结论

这批结果的主要失分不是“页面完全生成不出来”，而是更细的视觉还原问题：

1. **页面骨架通常正确**：三组的 layout 分都明显高于 style/asset 分，说明模型大多能抓住页面结构、模块层级和主要文本。
2. **样式系统不够精确**：颜色、字号、边框、间距、按钮状态、侧边栏背景、页脚/导航细节经常被近似化。
3. **真实视觉资产是最大短板**：FullFront 最明显，hero 图、产品图、logo、图标经常变成占位图、错误类别图或风格不一致的替代图。
4. **交互任务目前只评了静态初始图**：FullFront 的 Interaction Authoring 样本这次没有执行 click/hover 后再截图，因此结论只覆盖静态视觉还原，不覆盖交互实现质量。
5. **VLM judge 偏宽容**：一些肉眼可见的 viewport、比例、资产错误仍被打到 80+，所以这些分数适合做筛查和排序，不适合作为最终官方分数。

## 分组统计

| Benchmark | 平均分 | Layout | Content | Style | Asset | 主要短板 |
|---|---:|---:|---:|---:|---:|---|
| design2code | 92.9 | 96.9 | 98.0 | 88.7 | 94.0 | 主题色、边栏/header、局部组件样式 |

## 共性失分原因

### 1. 视觉资产替代失败

最典型出现在 `fullfront`。目标图里有真实图片、品牌 logo、产品图、背景图时，模型经常只能生成：

- 模糊渐变占位图；
- 错误类别或错误内容的图片；
- 颜色/形状不匹配的 logo；
- 缺少图标语义的简化符号；
- 和原页面风格不一致的图片裁切或遮罩。

代表样本：

- `fullfront / Interaction_Authoring_59_click_6_6`：主图从真实纹理图变成渐变占位，logo 颜色错误，得分 78。
- `fullfront / Text_to_code_14_new-img-code_127`：背景图、logo、关键事实图标失真，得分 82。
- `fullfront / Image_to_code_175_new-img-code_521`：产品图和图标资产偏差明显，得分 82。

### 2. 样式细节被“默认化”

模型常能生成相同区域，但样式不像原网页：

- 主题色不准确；
- button/tag/chip 的颜色、边框、圆角不一致；
- header/sidebar/footer 背景色缺失或错误；
- 输入框边框、阴影、分隔线变浅或变粗；
- 激活态、hover 态、下划线等状态样式缺失。

代表样本：

- `design2code / 378`：左侧导航栏和顶部 header 背景色不对，得分 88。
- `design2code / 229`：CTA、分享组件、文本颜色与目标不一致，得分 88。
- `flame / 17`：表单背景、输入框边框颜色、控件间距不一致，得分 88。

### 3. 尺寸、比例和 viewport 还原不稳定

部分生成图结构正确，但页面宽度、高度、卡片比例或内容密度和目标图不同：

- 内容整体更窄或更宽；
- 卡片留白过多；
- hero/主图高度错误；
- 长页面底部区域比例不对；
- 图表或列表区域被压缩。

这类问题 VLM judge 往往没有扣得很狠，但人工看会明显影响设计还原。

代表样本：

- `flame / 66`：图表比例和容器尺寸变化明显，但仍被打到 94。
- `fullfront / Image_to_code_79_new-img-code_173`：整体结构在，但主图、侧栏和页脚比例有偏差，得分 86。

### 4. 文本内容基本强，但状态文本/格式细节会错

文本整体是高分项，三组 content 平均都接近或超过 96。但还是有两类问题：

- 动态状态文本缺失或格式错误；
- 标签、按钮、状态说明被换行、改写或顺序错位。

代表样本：

- `flame / 38`：更新时间/动态内容不完整，得分 85。
- `flame / 74`：状态文本格式和单行/多行呈现不一致，得分 88。

## 和研究动机相关的判断

这批结果更像是在暴露一个方向：**当前模型已经能生成“看起来像网页”的骨架，但难以稳定生成“像目标设计系统”的精确前端实现**。

可以进一步拆成几个潜在研究点：

1. **资产感知与资产替换策略**：识别真实图片/Logo/图标类别、位置、裁切和风格，避免随意占位。
2. **设计 token 抽取**：从截图中显式提取颜色、字号、间距、圆角、阴影、边框，再约束生成代码。
3. **局部视觉差异反馈**：生成后用 VLM/视觉指标定位具体区域，再局部修复，而不是一次性生成。
4. **viewport/截图协议统一**：固定渲染尺寸、页面缩放和截屏高度，减少由于 viewport 不一致导致的误判。
5. **交互态单独评分**：FullFront/Vision2Web 这类任务需要执行 click/hover 后截图，否则只是在评静态初始态。

## 下一步建议

1. 对低分样本做人工标注，把失分原因归到固定标签：`资产错误`、`颜色错误`、`布局比例错误`、`文本错误`、`交互未验证`。
2. 在 FullFront interaction 样本上补一轮执行 click/hover 后的截图，再让 VLM 分别评初始态和交互态。
3. 对同一批样本再跑 CLIP/SSIM，观察它们和 VLM judge 对资产错误、布局错误的敏感性是否一致。
4. 如果要找论文动机，优先围绕 `资产/设计 token/局部修复` 三个方向继续扩展实验。
