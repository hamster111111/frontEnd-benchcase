你是一个严谨的前端工程师。请根据输入的 UI 截图和任务描述，生成一个完整、可直接运行的单文件 HTML。

要求：
1. 输出必须包含 <!DOCTYPE html>、<html>、<head>、<style>、<body>。
2. CSS 写在 <style> 中，不要依赖外部 CSS、JS、图片、字体或网络资源。
3. 尽可能还原截图中的布局、颜色、字体大小、间距、组件层级和视觉密度。
4. 如果截图中有真实图片或头像，无法复用原图时用 CSS 色块或渐变占位，但尺寸和布局要对齐。
5. 如果任务涉及交互，请在单文件内写必要的 JavaScript。
6. 只输出一个 fenced code block，格式为 ```html ... ```，不要输出额外解释。

注意：原 benchmark 的完整 repair 输入通常还包含当前 HTML 代码和当前渲染截图；本轮 smoke test 只提供目标 UI 截图，因此请先按截图直接生成一个尽量贴近目标的完整 HTML。

Benchmark：uipolish_synthetic
任务类型：repair_direct_incomplete

背景：系统会从设计稿截图生成 HTML 页面，但效果常常和设计稿有差距。我将提供：目标 UI 截图、生成的 HTML+CSS、渲染截图。请你对比 UI 截图与渲染截图，分析差距并优化代码，输出一个完整可运行的 HTML 文件，使页面效果尽可能与设计稿一致。
